# AdForge — Template Library

> **Source of truth for all 60 ad templates.**  
> Loaded at runtime by `src/lib/templates.ts` and injected into the agent system prompt.  
> Each template follows the same schema. Do not reorder fields.

---

## Schema Reference

| Field | Type | Description |
|---|---|---|
| `id` | integer 1–60 | Unique identifier |
| `category` | A–F | Template category group |
| `name` | string | Short human-readable name |
| `strategy_note` | string | When and why to use this template |
| `formats` | string[] | Supported aspect ratios |
| `recommended_format` | string | Best-performing default format |
| `variables` | object[] | All bracketed variables with fill type |
| `prompt` | string | Full Nano Banana 2 prompt with [BRACKETED] variables |

### Variable fill types
- `brand_dna` — auto-filled silently from Brand DNA (no user interaction)
- `ai_copy` — generated by Gemini 3 Flash, shown to user for approval before generation
- `user_input` — must be asked from user conversationally

---

## Category A — Performance / Direct Conversion

> Templates designed to drive immediate action. Lead with the offer, the product, or the claim. Highest-volume testing formats.

```yaml

strategy_note: Tests text rendering. If this comes back clean, the model handles copy. The simplest and most versatile performance template — start here for any new brand.
strategy_note_pt: "Testa a renderização do texto. O modelo domina a cópia se a saída for limpa. Simples e versátil: o ponto de partida para qualquer marca."

variables:
  - key: BACKGROUND
    fill: brand_dna
    note: Derived from primary_color or secondary_color in Brand DNA
  - key: YOUR HEADLINE
    fill: ai_copy
    note: Under 10 words. Direct benefit or bold claim.
  - key: YOUR SUBHEAD
    fill: ai_copy
    note: One sentence expanding the headline.
  - key: YOUR PRODUCT
    fill: brand_dna
    note: Full product description from Brand DNA
  - key: DETAILS
    fill: brand_dna
    note: Surface details, e.g. "casting a soft shadow on a marble surface"
  - key: BRAND
    fill: brand_dna
```

**Prompt:**
```

Top third: large bold sans-serif headline reading "[YOUR HEADLINE, under 10 words]". Below in smaller text: "[YOUR SUBHEAD, one sentence]". Bottom half: [YOUR PRODUCT] on the surface with [DETAILS]. Shot at 50mm f/2.8 from slightly above. [BRAND] logo bottom right. Clean, authoritative. [FORMAT] aspect ratio.
```

## Notes for Developers

- The `[FORMAT]` placeholder in every prompt is replaced at runtime with the user's selected aspect ratio string — e.g. `4:5`, `9:16`, `1:1`, or `16:9`.
- The `image_generation_modifier` from the Brand DNA is **prepended** to every prompt before the format line.
- All `brand_dna` variables are filled silently without user interaction.
- All `ai_copy` variables are generated by Gemini 3 Flash and shown to the user for one-click approval or edit before image generation.
- All `user_input` variables trigger a conversational question from the agent.
- Template parsing is handled by `src/lib/templates.ts`. The parser reads between the ` ```yaml ` and ` ``` ` fences for metadata, and between the ` ```  ` prompt fences for the prompt string.
